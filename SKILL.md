# Sticker Kit（微信表情包合辑助手）

本 Skill 用于指导和自动化生成微信表情包开放平台所需的各类素材，以及配套的合辑文案。它严格遵循官方的尺寸、格式、体积及设计规范。**表情图、横幅、封面、图标及赞赏引导图默认使用透明背景**（JPG 格式因不支持透明通道，自动回退为白色背景）。**赞赏致谢图须使用非透明的彩色背景**，不可使用透明背景或白色/接近白色的背景。

## 工作流程总览

1. **文案生成**：根据表情形象特点，生成合辑名称、合辑介绍和赞赏引导语
2. **资产准备**：确认主体形象素材，动态表情准备循环源文件
3. **素材生成**：使用内置脚本处理各类图片素材
4. **设计自检**：按清单逐项核查规范合规性

## 一、合辑文案生成

### 合辑名称

| 规则 | 说明 |
| :--- | :--- |
| 字数上限 | 不超过 8 个汉字，5 个汉字以内显示效果最佳 |
| 标点 | 不包含任何标点符号 |
| 空格 | 中文名称不出现空格 |

**示例：**

| 表情形象 | 合辑名称 |
| :--- | :--- |
| 慵懒的橘猫 | 橘猫摸鱼记 |
| 元气满满的小熊 | 元气小熊 |
| 毒舌打工人 | 打工人日常 |

### 合辑介绍

| 规则 | 说明 |
| :--- | :--- |
| 字数上限 | 不超过 80 个汉字 |
| 内容要求 | 充分展现表情的形象特点或故事情节 |

**示例：**

> 一只永远睡不醒的橘猫，用最慵懒的姿态诠释打工人的日常心声。摸鱼、发呆、假装努力……每一个表情都是你内心深处的真实写照。

### 赞赏引导语

| 规则 | 说明 |
| :--- | :--- |
| 字数范围 | 5～15 个汉字 |
| 风格要求 | 诙谐幽默，与表情主题有关联性，能有效引导用户发赞赏 |

**示例：**

| 表情主题 | 赞赏引导语 |
| :--- | :--- |
| 橘猫摸鱼 | 投喂一下摸鱼的猫 |
| 元气小熊 | 给小熊加点能量吧 |
| 打工人日常 | 赏杯奶茶续命吧 |
| 可爱兔子 | 兔兔这么可爱快赞赏 |

### 文案生成步骤

1. 了解表情包的主体形象、性格特点和核心情绪。
2. 提炼 2～3 个候选合辑名称，优先选择 5 字以内、朗朗上口的方案。
3. 撰写合辑介绍，突出形象辨识度和情感共鸣点，控制在 80 字以内。
4. 生成 3～5 条赞赏引导语候选，选取最幽默且与主题关联最强的一条。

## 二、素材规范概览

| 素材类型 | 格式 | 尺寸 (px) | 体积限制 | 数量 | 核心设计要求 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **表情图** | GIF/PNG | 240 × 240 | < 500 KB | 8–24 | 动态须循环、流畅；布局紧凑，少留白；**透明背景** |
| **横幅图** | PNG | 750 × 400 | < 500 KB | 1 | **无文字**；色调明朗；具故事性；**透明背景** |
| **封面图** | PNG | 240 × 240 | < 500 KB | 1 | **透明背景**；正面半身/全身像；无白色描边/锯齿 |
| **图标** | PNG | 50 × 50 | < 100 KB | 1 | **透明背景**；头部正面；无白色描边/锯齿；无方框 |
| **赞赏引导** | GIF/PNG | 750 × 560 | < 500 KB | 1 | 风格一致；引导用户赞赏；**透明背景**；**避免白色或接近白色的背景/边缘** |
| **赞赏致谢** | GIF/PNG | 750 × 750 | < 500 KB | 1 | 风格一致；激发分享意愿；**禁止透明背景**；**须使用彩色不透明背景**；**避免白色或接近白色的背景/边缘** |

> **注意**：JPG 格式不支持透明通道，脚本会自动回退为白色背景。建议所有素材优先使用 PNG 格式。赞赏致谢图须使用彩色不透明背景，生成时应将 `transparent` 参数设为 `False` 并指定背景色。

## 三、素材生成

使用内置脚本 `scripts/process_sticker.py` 进行自动化处理，确保尺寸和体积达标。`transparent` 参数默认为 `True`，无需显式传入。

```bash
# 生成静态表情图 (240×240, PNG, 透明背景 — 默认)
python3 /home/ubuntu/skills/sticker-kit/scripts/process_sticker.py input_static.png static_sticker.png 240 240 PNG 500

# 生成动态表情图 (240×240, GIF, 循环播放, 透明背景 — 默认)
python3 /home/ubuntu/skills/sticker-kit/scripts/process_sticker.py input_animated.gif animated_sticker.gif 240 240 GIF 500 True True True

# 生成封面图 (240×240, PNG, 透明背景 — 默认)
python3 /home/ubuntu/skills/sticker-kit/scripts/process_sticker.py input.png cover.png 240 240 PNG 500

# 生成图标 (50×50, PNG, 透明背景 — 默认)
python3 /home/ubuntu/skills/sticker-kit/scripts/process_sticker.py head.png icon.png 50 50 PNG 100

# 生成赞赏致谢图 (750×750, PNG, 彩色不透明背景 — 必须关闭透明)
python3 /home/ubuntu/skills/sticker-kit/scripts/process_sticker.py thanks.png thanks_out.png 750 750 PNG 500 False
```

**参数顺序：** `input_path`, `output_path`, `width`, `height`, `format`, `max_kb`, `[transparent=True]`, `[is_animated=False]`, `[loop_gif=False]`

- `transparent`：默认 `True`；JPG 格式时自动忽略，回退为白色背景；赞赏致谢图须设为 `False`。
- `is_animated`：设为 `True` 时处理动态 GIF。
- `loop_gif`：仅在 `is_animated=True` 且 `format=GIF` 时有效，`True` 表示循环播放。
- 表情图数量可在 8–24 张之间灵活指定。

## 四、设计自检清单

### 通用要求

- **严禁拉伸**：所有图片必须保持原比例，严禁因拉伸或压扁导致变形。
- **留白控制**：合理安排布局，避免过多无意义留白。
- **背景处理**：所有素材**默认透明背景**；避免使用白色背景（防止与微信底色混淆）。

### 特定要求

- **横幅图**：绝对不能出现文字；色调活泼，与微信底色有区分。
- **封面/图标**：形象不应有白色描边，边缘需平滑无锯齿。
- **赞赏引导图**：内容必须与表情主题高度相关，风格统一；**避免白色或接近白色的背景/边缘**。
- **赞赏致谢图**：内容必须与表情主题高度相关，风格统一；**禁止使用透明背景**；**须使用彩色不透明背景**；**避免白色或接近白色的背景/边缘**。

---

## 五、错误处理

- 图片体积超标：脚本会自动尝试降低质量；若仍超标，请简化画面或减少 GIF 帧数。
- 出现锯齿：检查源文件分辨率或增加抗锯齿处理。
- GIF 透明区域显示异常：检查源文件是否包含正确的 Alpha 通道。
